# KcalSnap Landing MVP

Учебный MVP-лендинг для проекта **подсчёта калорий по фото**.

## Что требовалось по презентации

Согласно заданию, нужно было:
- сделать одностраничный лендинг;
- использовать стек `HTML + CSS + JS`;
- добавить заголовок, описание, CTA, блок «как это работает» и форму ввода данных;
- реализовать логику, где пользователь вводит данные, нажимает кнопку и получает ответ;
- сохранить вопрос и ответ в `localStorage` и показывать их после обновления страницы;
- добавить **одну интерактивную функцию** и описать изменения логики в документации.

## Что реализовано

### 1. Лендинг
На странице есть:
- hero-блок с ценностным предложением;
- CTA-кнопки;
- блок «Как это работает»;
- demo-форма для ввода блюда;
- блок с результатом;
- история запросов;
- дополнительная интерактивная функция.

### 2. Основная логика формы
Пользователь вводит текстовое описание блюда, например:
> «куриное филе, рис и салат»

После нажатия на кнопку:
- JavaScript анализирует текст;
- локально рассчитывает примерные калории и БЖУ;
- показывает ответ в интерфейсе.

Это **имитация AI-ответа для MVP**. В рабочей версии сюда можно подключить реальный AI API.

### 3. Сохранение в localStorage
После получения ответа сохраняются:
- вопрос пользователя;
- расчёт калорий и БЖУ;
- дата и время.

При перезагрузке страницы:
- история снова подгружается;
- последний ответ показывается в блоке результата.

Ключ localStorage:
```js
kcalsnap_history_v1
```

### 4. Дополнительная интерактивная функция
Для домашнего задания добавлен **калькулятор дневной нормы калорий**.

Пользователь выбирает:
- пол,
- цель,
- возраст,
- вес,
- рост,
- активность.

После нажатия на кнопку рассчитывается примерная суточная норма калорий.

### 5. Изменения логики, которые нужно отразить в GitHub
В рамках домашней работы добавлена новая логика:
1. отдельная форма калькулятора нормы калорий;
2. расчёт по базовой формуле BMR + коэффициент активности;
3. поправка на цель: похудение / поддержание / набор массы;
4. вывод результата в интерфейсе без перезагрузки страницы.

## Как проверить

1. Открыть `index.html` в браузере.
2. Прокрутить до блока **«Демо AI-анализа блюда»**.
3. Ввести описание блюда.
4. Нажать **«Получить ответ AI»**.
5. Обновить страницу — история должна сохраниться.
6. Открыть DevTools → Application → Local Storage.
7. Проверить, что ключ `kcalsnap_history_v1` содержит данные.
