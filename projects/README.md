# KcalSnap Landing MVP

Учебный MVP-лендинг для проекта **подсчёта калорий по фото, тексту и голосу**.

---

## Что делает приложение

KcalSnap позволяет пользователю ввести блюдо тремя способами — текстом, голосом или через загрузку фото — и мгновенно получить расчёт калорий, белков, жиров и углеводов. Все результаты сохраняются в истории через localStorage.

---

## Добавленные функции

### 1. Три способа ввода (вкладки)

**Текст → Голос → Фото** — переключаются кнопками над формой.

#### Текстовый ввод
- Пользователь вводит продукты с граммовкой.
- Поддерживаемые форматы:
  - `рис 100 г, курица 170 г`
  - `банан 120 г`
  - `яйца 2, хлеб 50 г`
- Несколько ингредиентов разделяются запятой или точкой с запятой.

#### Голосовой ввод
- Используется браузерный **Web Speech API** (`SpeechRecognition` / `webkitSpeechRecognition`).
- Пользователь нажимает кнопку-микрофон, говорит название блюда и граммовку.
- Распознанный текст автоматически появляется в поле ввода.
- Затем нажимает «Рассчитать» — расчёт идёт по тому же механизму, что и текстовый ввод.
- Если браузер **не поддерживает** Web Speech API, кнопка блокируется и показывается сообщение: поддерживаются Chrome и Edge.

#### Фото-ввод (MVP, мок)
- Пользователь загружает фото блюда.
- Показывается превью изображения.
- Вместо реального AI-анализа отображается список из 5 мок-блюд:
  - 🍚 Рис с курицей
  - 🍝 Паста
  - 🥗 Салат
  - 🍳 Яичница с тостами
  - 🌾 Овсянка с бананом
- Пользователь выбирает подходящее — система подставляет ингредиенты и считает калории.
- **Распознавание фото — демо-заглушка для MVP.** В рабочей версии здесь подключается AI API (например, Google Vision или OpenAI Vision).

### 3. Расчёт нутриентов

Для каждого распознанного ингредиента система считает:
- Калории
- Белки (г)
- Жиры (г)
- Углеводы (г)

Выводятся отдельные строки по каждому продукту и итоговая сумма в виде карточек.

### 4. История запросов

Каждый результат сохраняется в `localStorage` (ключ: `kcalsnap_history_v1`) в следующем формате:

```json
{
  "inputType": "text",
  "originalInput": "рис 150 г, курица 120 г",
  "parsedIngredients": [...],
  "calories": 393,
  "protein": 41.0,
  "fat": 4.9,
  "carbs": 42.3,
  "timestamp": "2025-01-01T12:00:00.000Z"
}
```

- `inputType`: `text` | `voice` | `photo`
- История загружается после обновления страницы
- Хранится до 20 последних записей
- Кнопка «Очистить историю» удаляет все записи

### 5. Калькулятор дневной нормы

Отдельная форма для расчёта суточной потребности в калориях по формуле Миффлина–Сан Жеора + коэффициент активности + поправка на цель.

## Как проверить

### Текстовый ввод
1. Откройте вкладку **📝 Текст**.
2. Введите: `рис 150 г, курица 120 г`.
3. Нажмите **Рассчитать**.
4. Должен появиться результат: ≈ 393 ккал, Б 41 г, Ж 4.9 г, У 42 г.

### Голосовой ввод
1. Откройте в Chrome или Edge.
2. Перейдите во вкладку **🎙 Голос**.
3. Нажмите кнопку-микрофон.
4. Разрешите доступ к микрофону.
5. Скажите: «рис сто пятьдесят грамм, курица сто двадцать грамм».
6. Нажмите **Рассчитать**.

### Фото-ввод
1. Перейдите во вкладку **📷 Фото**.
2. Загрузите любое фото блюда.
3. Выберите блюдо из предложенного списка.
4. Результат появится справа, запись попадёт в историю.

### LocalStorage
- Откройте DevTools → Application → Local Storage.
- Ключ: `kcalsnap_history_v1`.
- После обновления страницы история сохраняется.
